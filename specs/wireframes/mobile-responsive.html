Responsive Design Patterns

Complete summary of responsive behavior and implementation considerations for AgentPane mobile-responsive design.

Mobile-First Strategy
  • Start simple on mobile: Single column, bottom nav, stacked cards
  • Progressive enhancement: Add complexity as viewport grows (sidebar, columns, panels)
  • Viewport meta tag: <meta name="viewport" content="width=device-width, initial-scale=1.0">
  • Flexible containers: Use max-width constraints, not fixed widths (except for grid columns)
  • Relative units: Prefer rem/em over px for scalability
Touch Interaction Enhancements
  • Active state feedback: Visual response to touch with :active pseudo-class (background color change)
  • No hover on touch: Hover states only on desktop via @media (hover: hover)
  • Gesture support: Swipe left/right for navigation, long-press for context menu, pull-to-refresh
  • Reduced motion: Respect prefers-reduced-motion media query for animations
  • Haptic feedback: Brief vibration on action confirmation (native mobile feature)
Responsive Layout Breakpoints
  • sm (640px): Hamburger ‚Üí Sidebar, Bottom nav primary, Stacked cards
  • md (768px): Sidebar drawer on demand, 2-column kanban, Touch-optimized
  • lg (1024px): Persistent sidebar, 3-column kanban, Detail panels appear
  • xl (1280px): 4+ columns, split-view details, Maximum density
  • Fluid between: Use clamp() to scale values smoothly without hard jumps
Performance Considerations
  • Mobile optimization: Lazy load columns on kanban board, reduce card animations on low-end devices
  • Touch scrolling: -webkit-overflow-scrolling: touch for momentum scrolling on iOS
  • Viewport optimization: Avoid layout thrashing with CSS-only responsive changes
  • Image scaling: Use srcset for responsive images, avoid oversized assets on mobile
  • CSS containment: Use contain: layout on scrollable sections for performance
Accessibility Requirements
  • Minimum touch target: 44√ó44px for all interactive elements (WCAG AAA)
  • Focus indicators: Visible keyboard focus on all buttons and inputs
  • Color contrast: 4.5:1 minimum for text, 3:1 for icons (WCAG AA)
  • Motion preferences: Disable animations if user prefers reduced motion
  • Semantic HTML: Use proper heading hierarchy, nav, main, footer landmarks
  • Screen reader support: ARIA labels for icon buttons, alt text for images
  • Keyboard navigation: All features accessible via Tab and Enter keys
