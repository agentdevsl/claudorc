Design Notes

Worktree Lifecycle

  • Created when an agent starts work on a branch
  • Active while agent is assigned or recent activity detected
  • Stale after inactivity period or branch deletion
  • Pruned via manual or automatic cleanup
  • Main worktree is always protected

Auto-Cleanup Policies

  • Default inactivity threshold: 7 days
  • Immediate cleanup if remote branch deleted
  • Disk usage alerts at 80% capacity
  • Protected worktrees skip auto-cleanup
  • Configurable per-repository thresholds

Branch-Worktree Mapping

  • One worktree per branch (enforced by Git)
  • Path derived from: repo/branch-slug
  • Branch names sanitized for filesystem
  • Mapping stored in .git/worktrees/
  • Agents lookup worktree by branch name
