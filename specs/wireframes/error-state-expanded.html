Design Notes

Error Visualization Priorities

  • Error banner immediately visible on load
  • Error type and message prominently displayed
  • Stack trace collapsible but expanded by default
  • Color-coded log entries for quick scanning
  • File paths clickable to view source context

Retry Strategies

  • Feedback textarea for contextual guidance
  • Checkpoint recovery reduces wasted compute
  • Model upgrade option for complex failures
  • Turn limit increase for scope expansion

Checkpoint Recovery

  • Checkpoints saved after each successful action
  • Resume from last known good state
  • Preserves file edits and git state
  • Reduces retry time by ~70% on average
