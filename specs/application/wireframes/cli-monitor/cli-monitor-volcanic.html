Design Notes: Volcanic Observatory

Concept The earth's geological cross-section becomes the monitoring system. Each layer of the earth maps to a level of abstraction in the agent monitoring hierarchy.

  • Tectonic Plates = Projects — The crust is divided into labeled plates with glowing fault lines at boundaries. Plate size reflects project scope.
  • Magma Chambers = Sessions — Glowing caverns beneath each plate. Working sessions pulse with convection particles; waiting sessions show cooling crust forming; idle sessions are cooled rock with residual heat.
  • Eruptions = Completions — When sessions complete, ash clouds and lava projectiles burst from the surface. The eruption count in the HUD tracks completed tasks.
  • Lava Flows = Dependencies — Animated particle streams between chambers represent shared data flows. Speed indicates throughput.
  • Geological Layers — Surface (projects), Upper Mantle (sessions), Lower Mantle (tokens), Outer Core (aggregate metrics), Inner Core (system health with green/red glow).

Seismograph Each session has a live trace. Working = high-amplitude jagged waves. Waiting = periodic tremors. Idle = flat baseline. P-wave markers = tool starts, S-wave markers = tool results. Richter magnitude reflects overall system activity.

Minerals Token types are geological deposits: Obsidian (input), Quartz (output), Amber (cached). Cost is "mineral value" in gold. Rate limit is "tectonic pressure."
