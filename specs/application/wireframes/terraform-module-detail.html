terraform-aws-vpc

A Terraform module for creating a production-grade AWS VPC with support for multiple availability zones, public and private subnets, NAT gateways, and VPC flow logs.

Usage

module "vpc" {
  source  = "app.terraform.io/acme-infrastructure/vpc/aws"
  version = "3.2.1"

  vpc_cidr           = "10.0.0.0/16"
  availability_zones = ["us-east-1a", "us-east-1b", "us-east-1c"]
  private_subnets    = ["10.0.1.0/24", "10.0.2.0/24", "10.0.3.0/24"]
  public_subnets     = ["10.0.101.0/24", "10.0.102.0/24", "10.0.103.0/24"]

  enable_nat_gateway = true
  enable_flow_logs   = true
}

Requirements

Terraform >= 1.5.0 and AWS provider >= 5.0 are required. Ensure your AWS credentials have permissions to create VPC resources, NAT gateways, and CloudWatch log groups.

Notes

When single_nat_gateway is set to false, a NAT gateway is created in each availability zone for high availability. This increases cost but ensures private subnet connectivity survives an AZ failure.
