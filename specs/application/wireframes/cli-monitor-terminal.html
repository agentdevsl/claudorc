Design Notes — Terminal Panes Variation

Multi-Pane Terminal Concept

This variation replaces the card-based session list with a tmux-style terminal grid. The entire main area is devoted to simultaneous live streams from multiple Claude Code sessions, maximizing the amount of real-time output visible at once.

  • 2x2 grid shows 4 sessions simultaneously (configurable via header button)
  • Each pane is a full terminal with scrollable output
  • Compact tab bar per pane shows session ID, branch, project, and turn count
  • Panes can be maximized on hover for focused viewing

Visual State Indicators

  • Active panes have a glowBorder animation with subtle green glow
  • Waiting panes show an amber border tint
  • Idle panes are dimmed to 55% opacity, brighten on hover
  • Status dots use pulse animation for working sessions
  • Active terminals show a blinking block cursor at the output position
  • New lines fade in with fadeInLine animation

Color Coding System

Terminal output uses a consistent color scheme across all panes:

  • [user] — Blue (accent) for user messages
  • [claude] — Purple (done) for Claude responses
  • [tool:*] — Amber (attention) for tool invocations, with left-border highlight
  • [system] — Gray (subtle) for system messages
  • file paths — Green (success) for file references

Layout Efficiency

  • Status bar replaces the top monitor cards, condensing aggregate stats into a single line
  • Session picker floats in the bottom-right, allowing pane reassignment without leaving the terminal view
  • Unassigned sessions appear below the divider in the picker
  • Header is compact (reduced padding) to maximize terminal area
  • Grid layout uses 1px gap with emphasis background to create tmux-style dividers
